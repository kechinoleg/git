1.Как отказаться от изменений в файле (файлах) находящийся в рабочем 
каталоге (work directory)?

Ответ: 
Отмена всех изменений в файлах: git checkout -- .
Отмена изменений в определенных файлах: git checkout -- file1.txt 
file2.txt
Отмена изменений и добавление их в стэйдж: git reset HEAD .
Отмена изменений и удаление из истории коммитов: git reset --hard HEAD^

2.Как вернуть файл из индекса в рабочий каталог?

Ответ:
git restore --staged <имя_файла>
Женуть все файлы git restore --staged .

3.Как отменить ошибочный commit ? - Перечислить все возможные варианты.

Ответ:
Отмена последнего коммита без изменения истории: git add <изменения>
git commit --amend
Отмена коммита и изменение истории:
git rebase -i HEAD~<количество коммитов начиная с последнего>
Отмена коммита и создание нового коммита: git revert <хэш-код коммита>
Сброс до предыдущего коммита: git reset --hard HEAD^
Исправление с использованием revert: git revert -n <хеш код коммита>
git commit -m «Коментарий <хеш код коммита>"

4.Как исправить коммит отправленный на сервер GitHub?

Ответ:
Исправление последнего коммита с изменением истории: 
git add <изменения>
git commit --amend
git push --force origin <моя_ветка>
Исправление старых коммитов с изменением истории: git rebase -i 
HEAD~<количество коммитов начиная с последнего>
Исправление с использованием revert: 
git revert <хеш код коммита>
git push origin <моя_ветка>

5.Как создать новую ветку и переключится в нее?

Ответ:
git branch <имя_новой_ветки>
git checkout <имя_ветки>
